# 进销存系统 v1.3 需求文档

## 核心主题
规则驱动的智能功能 — 零资源消耗，纯代码实现

## 项目路径
~/v5/test/inventory-app/

## 技术栈
- 前端: React 19 + TypeScript + Ant Design
- 后端: Tauri 2 (Rust) + SQLite
- 前端入口: src/pages/
- 后端入口: src-tauri/src/db.rs (数据库), src-tauri/src/lib.rs (Tauri命令)

## 需求清单

### 1. 智能补全 — 商品添加/编辑时自动推荐
**场景**: 添加或编辑商品时，输入商品名称后自动推荐分类、单位、价格
**后端**: 
- db.rs 添加 `get_product_suggestions(name: String)` 方法
- 从已有商品中模糊匹配名称相似的商品，返回其 category, unit, cost_price, sell_price
- 使用 LIKE 模糊匹配，取最近更新的3条
**前端**:
- Products.tsx 的商品表单中，商品名称输入框添加 onSearch 事件
- 用户输入时调用后端接口获取推荐
- 以 Popover 或下拉列表形式展示推荐值
- 用户点击推荐项可一键填充

### 2. 销售趋势 — Dashboard 新增趋势卡片
**场景**: 数据概览页展示近7天销售趋势和同比数据
**后端**:
- db.rs 添加 `get_sales_trend()` 方法
- 返回最近7天每天的销售额 (按天分组统计 outbound_orders.total)
- 返回上月同期7天的销售额 (用于对比)
- 返回格式: { daily: [{date, amount}], last_period_daily: [{date, amount}] }
**前端**:
- 新建 src/components/SalesTrend.tsx 组件
- 使用 Ant Design 的简单柱状图展示（不用引入图表库，用 CSS 实现）
- 用 div + flex 实现简易柱状图，柱子高度按最大值归一化
- Dashboard 页面引入此组件

### 3. 滞销识别 — 标记长期无销量商品
**场景**: 识别30天内没有出库记录的商品，提醒用户关注
**后端**:
- db.rs 添加 `get_slow_moving_products(days: i32)` 方法
- 查询 products 中在指定天数内没有 outbound_orders 记录的商品
- 返回商品信息 + 最后出库日期
**前端**:
- Dashboard 新增"滞销商品"卡片（黄色警告样式）
- 显示滞销商品数量和列表
- 列表项显示：商品名、最后出库日期、当前库存

### 4. 经营周报 — 自动生成文本报告
**场景**: 一键生成本周/本月经营数据报告
**后端**:
- db.rs 添加 `get_weekly_report()` 方法
- 返回本周数据: 总销售额、总采购额、利润、新增商品数、入库/出库单数
- 返回上周数据（用于对比计算增减百分比）
- 返回格式: { current: {...}, previous: {...} }
**前端**:
- 新建 src/pages/Report.tsx 页面
- 侧边栏添加"经营报告"菜单项（图标 FileTextOutlined）
- 页面展示卡片式报告：
  - 销售额（本周 vs 上周，增减百分比）
  - 采购额
  - 利润
  - 入库/出库单数
- 添加"导出报告"按钮，导出为 txt 文件
- 支持切换周报/月报

### 5. 智能搜索 — 全局搜索增强
**场景**: 商品搜索支持拼音首字母匹配
**前端**:
- 新建 src/utils/pinyin.ts — 简单的汉字转拼音首字母工具函数
- 不引入第三方库，使用内嵌的常用汉字拼音映射表（覆盖3000常用字即可）
- Products.tsx 的搜索逻辑增强：同时匹配原名和拼音首字母
**示例**:
- 输入 "sj" 匹配 "手机"
- 输入 "shouji" 匹配 "手机"
- 输入 "苹果" 直接匹配 "苹果"

## 注意事项
- 所有后端方法返回 JSON 字符串
- 前端使用 invoke 调用 Tauri 命令
- 保持现有代码风格
- 每完成一个功能点，确保 npm run build 和 cargo check 通过
- 不引入新的 npm 依赖（图表用 CSS 实现，拼音用内置映射表）
