# 进销存系统 v1.2 需求文档

## 项目路径
~/v5/test/inventory-app/

## 技术栈
- 前端: React 19 + TypeScript + Ant Design
- 后端: Tauri 2 (Rust) + SQLite
- 前端入口: src/pages/
- 后端入口: src-tauri/src/db.rs (数据库), src-tauri/src/lib.rs (Tauri命令)

## 需求清单

### 1. 修改密码功能
**位置**: 在侧边栏底部"退出登录"上方添加"修改密码"菜单项
**页面**: 新建 src/pages/ChangePassword.tsx
**后端**: 
- db.rs 添加 `change_password(old_hash, new_hash)` 方法
- lib.rs 添加 `change_password` Tauri 命令
**逻辑**:
- 验证旧密码 → 设置新密码
- 成功后提示"密码修改成功"，不退出登录
**UI**: 简单表单，旧密码 + 新密码 + 确认新密码

### 2. 入库/出库记录日期筛选
**位置**: 在入库页面和出库页面的记录列表上方
**组件**: 使用 Ant Design 的 DatePicker.RangePicker
**后端**: 修改 get_inbound_records 和 get_outbound_records，接受可选的 start_date/end_date 参数
**SQL**: 添加 WHERE created_at BETWEEN ? AND ? 条件
**默认**: 不选日期时显示最近100条（现有逻辑）

### 3. 数据导出 Excel
**依赖**: 添加 xlsx 包 (npm install xlsx)
**位置**: 在入库记录和出库记录表格右上方添加"导出"按钮
**功能**: 导出当前筛选结果为 .xlsx 文件
**列**: 与表格列一致
**文件名**: 入库记录_2026-04-12.xlsx / 出库记录_2026-04-12.xlsx

### 4. 库存预警
**位置**: 数据概览页(Dashboard)新增预警卡片
**逻辑**: 库存 < 10 的商品为预警商品（沿用现有阈值）
**后端**: db.rs 添加 `get_low_stock_products()` 方法，返回 stock < 10 的商品列表
**UI**: Dashboard 底部添加一个 Alert 或 Card，显示预警商品数量和列表
**样式**: 红色警告样式

## 注意事项
- 所有后端方法返回 JSON 字符串
- 前端使用 invoke 调用 Tauri 命令
- 保持现有代码风格
- 每完成一个功能点，确保 `npm run build` 和 `cargo check` 通过
